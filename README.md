# Nexus AI Bot — Event Management System

B2B SaaS-платформа (Telegram бот) для интеллектуального сорсинга и управления наймом на мероприятия. Предназначена для рекрутеров и кандидатов.

## 🌟 Возможности

### Для рекрутеров:
- **Управление мероприятиями**: Создание ивентов, задание лимитов, требований по полу и местах проведения.
- **Опросы в группах**: Автоматическая публикация опросов в привязанные Telegram-группы.
- **Интерактивный отбор**: Возможность выбрать конкретных кандидатов из тех, кто проголосовал "Да" или "Возможно".
- **Назначение смен**: Установка времени прихода и ухода для каждого выбранного кандидата.
- **Массовые рассылки**: Команда `/announce` для рассылки сообщений участникам конкретного мероприятия.
- **Google Таблицы**: Генерация Excel/Google-таблиц с авто-подсчетом отработанных часов и расчетом выплат.
- **Напоминания**: Автоматические уведомления кандидатам за день до мероприятия (Cron job) и напоминания об оплате для рекрутеров (через 14 дней).
- **Аудит логгирование**: Запись всех действий по мероприятию в базу данных для отслеживания изменений.

### Для кандидатов:
- Удобная регистрация через Telegram-бота (выбор пола, отправка контакта).
- Голосование в опросах группы.
- Получение индивидуальных уведомлений с деталями смены.
- Возможность подтвердить или отклонить свое участие прямо в боте.

## 🛠 Технологический стек
- **Язык**: Python 3.10+
- **Фреймворк для бота**: `python-telegram-bot`
- **База данных**: Supabase (PostgreSQL) с использованием официального Python-клиента `supabase`
- **Планировщик задач**: `APScheduler` (Асинхронный cron)
- **Работа с Google Sheets**: `google-api-python-client`
- **Генерация Excel**: `openpyxl`
- **Валидация конфигурации**: `pydantic-settings`

## 🚀 Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repo-url>
   cd Nexus_for_waiters
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройка окружения:**
   Скопируйте пример окружения и заполните `.env` файл:
   - `BOT_TOKEN`: Токен вашего бота (от @BotFather)
   - `ADMIN_USER_IDS`: ID Telegram аккаунтов рекрутеров через запятую (напр. `123456,789012`)
   - `GROUP_CHAT_ID`: ID группы, куда будут отправляться опросы
   - `SUPABASE_URL` и `SUPABASE_KEY`: Данные для подключения к вашей БД Supabase
   - `GOOGLE_CREDENTIALS_FILE`: Путь к JSON-файлу сервисного аккаунта Google (для листов)

4. **База данных:**
   Выполните SQL скрипт из файла `supabase_schema.sql` в редакторе SQL вашего проекта Supabase, чтобы создать необходимые таблицы, индексы и RLS-политики.

5. **Запуск бота:**
   ```bash
   python main.py
   ```

## 📂 Структура проекта
- `main.py` — Точка входа в приложение, регистрация хендлеров и инициализация.
- `config.py` — Загрузка настроек через pydantic_settings.
- `database.py` — Singleton Supabase-клиент.
- `models/` — Pydantic модели и Enum'ы для типизации событий и кандидатов.
- `handlers/` — Роутинг и обработка команд Telegram (опросы, мероприятия, кандидаты, админка).
- `services/` — Бизнес-логика системы (работа с БД, Google Sheets, планировщиком задач, генерацией Excel, аудит-логами).
- `utils/` — Общие утилиты, генераторы клавиатур и валидаторы.

## 📜 Команды рекрутера

- `/create_event` — Пошаговое создание мероприятия.
- `/list_events` — Вывод активных мероприятий с панелью управления.
- `/publish_poll <event_id>` — Публикация опроса в привязанную группу.
- `/voters <event_id>` — Просмотр ответов на опрос.
- `/select_candidates <event_id>` — Интерактивный отбор кандидатов.
- `/set_times <event_id> <user_id> <HH:MM> <HH:MM>` — Назначение часов работы.
- `/notify_candidates <event_id>` — Отправка подтверждений.
- `/announce <event_id> <text>` — Массовая рассылка информации всем участникам.
- `/create_sheet <event_id>` — Формирование онлайн-таблицы в Google Sheets.
- `/export_excel <event_id>` — Выгрузка .xlsx файла.
- `/payment_reminder <event_id>` — Запланировать напоминание об оплате.
- `/logs <event_id>` — Аудит действий по мероприятию.
- `/close_event <event_id>` — Отправить в архив.
